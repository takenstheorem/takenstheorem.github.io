Shipyard Work Unit (VWU)

A community-built add-on project by Takens Theorem - an experimental ERC-20 token within The Vessel ecosystem that rewards when vessel payloads change.

This is an art project. An experiment of community activity and mutual support. VWU is assumed to have no value. Use at your own risk.

What is The Vessel?

The Vessel is a generative art and on-chain machine project created by stephensantoro_ and producedbydav where each token (a "craft") carries a dynamic payload that evolves through interactions with on-chain machines. Craft payloads can be crafted, mixed, and transformed via various contracts in the ecosystem. An extensive, expansive sociocultural machine.

Shipyard Work Units are a lightweight, experimental token that rewards vessel owners for provable payload evolution.

Visit thevessel.fun to explore the main project and learn more about vessels.

How VWU Works

  • Each successful claim of a changed payload mints exactly 1 VWU.
  • Claims are per-vessel and enforce a cooldown (configurable by the contract owner).
  • If a vessel has an associated machine, the claim is keyed off the machine's address; otherwise, it uses the vessel's payload hash.
  • First claim per vessel always succeeds (previous hash is zero).

Key Details

  • Name: The Vessel Shipyard Work Unit
  • Symbol: VWU
  • Decimals: 0 (integer-only by design)
  • Supply: Dynamic; grows with successful claims
  • Claim Cooldown: Configurable (default: 0 hours)

Interacting with VWU

Use the contract's claimWorkUnit(vesselTokenId) or claimWorkUnitBatch(vesselTokenIds[]) functions to mint tokens. Standard ERC-20 functions (transfer, approve, balanceOf, etc.) are supported.

Check a vessel's machine with checkMachine(vesselTokenId) to see whether claims will hash the payload or the machine address.

About This Contract

This is an experimental toy token within The Vessel ecosystem. Use at your own risk. The contract is self-contained and does not rely on external oracles for its core logic.
